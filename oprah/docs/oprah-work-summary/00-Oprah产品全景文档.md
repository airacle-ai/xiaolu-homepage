# Oprah 产品全景文档

> 从产品经理视角，完整记录 Oprah 从 0 到 1 的构建过程。涵盖产品定位、用户洞察、功能设计、技术决策、设计语言、增长策略和未来路线图。

---

## 一、产品愿景与定位

### 1.1 一句话定位

**AI 对话式人格探测 + 关系碰撞工具。** 用 AI 对话制造被看见感，用结构化维度建立可信度，用关系碰撞完成社交传播。

### 1.2 核心价值主张

用户完成三段旅程：

```
对话（被理解）→ 画像（被验证）→ 碰撞（被传播）
```

| 阶段 | 用户心理 | 产品提供 |
|------|---------|---------|
| 对话 | "这个 AI 真的在听我说话" | 回声机制、矛盾捕捉、自然追问 |
| 画像 | "原来我是这样的人，而且有证据" | 19 维结构化 + 三级证据层 |
| 碰撞 | "我要让朋友也看看" | 关系名片 + 热梗标签 + 社交传播 |

### 1.3 产品命名

- **曾用名：** 晴天 SunnySide（疗愈路线）
- **现用名：** Oprah（回归原名，暗黑潮流路线）
- **决策逻辑：** "晴天"过于柔美，与目标年轻用户群的审美不匹配。Oprah 中性、好记、有辨识度，适配暗黑赛博风格

### 1.4 五层价值模型

这是产品的完整价值体系，每一层解决一个核心问题：

| 层 | 回答的问题 | 关键机制 |
|----|-----------|---------|
| 1. 被理解感 | "它真的懂我吗？" | 回声机制、矛盾捕捉三层、去模板化追问 |
| 2. 可信度 | "它凭什么这么说？" | 结构化证据（摘要→引用→反例）、三级确信度标记 |
| 3. 传播 | "我怎么让朋友知道？" | 关系名片 Canvas 导出、热梗标签体系 |
| 4. 行动 | "知道之后我做什么？" | 行为建议卡片、反馈闭环 |
| 5. 留存 | "我为什么要回来？" | 时间轴演化、跨关系模式、反思触达 |

## 二、用户洞察

### 2.1 目标用户

| 用户类型 | 行为特征 | 产品策略 |
|---------|---------|---------|
| 好奇尝鲜者 | 被朋友分享吸引，想快速看到结果 | 随性漫游（3 分钟），立即满足 |
| 深度探索者 | 对自我认知有真实需求 | 深层剖视（10 分钟），四重思维框架 |
| 社交传播者 | 做完后想和朋友互动 | 碰撞功能 + 关系名片分享 |
| 日常陪伴者 | 想有一个可以随时聊天的对象 | 心情日记（未来）+ 轻量交互 |

### 2.2 用户耐心分层

不同类型用户的可接受对话轮数不同，产品做了显性分流：

```
登录 → 选模式
       ├─ 随性漫游（3 分钟 / 7-8 轮 / 12 核心维）
       └─ 深层剖视（10 分钟 / 10-15 轮 / 19 全维）
```

用户随时可通过进度条按钮提前结束对话查看结果。

### 2.3 新老用户差异

| | 新用户 | 老用户 |
|----|------|------|
| 开场 | 轻柔自我介绍 + 第一个情境题 | 像老朋友打招呼 + 提到上次身份标签 |
| 对话方向 | 自然流动，不按固定顺序 | "最近有变化吗？想继续深挖某个方向？" |
| 数据 | 全新 localStorage/Supabase | 恢复历史对话 + 已有维度进度 |

## 三、功能架构

### 3.1 页面结构

```
/login        PIN 登录页
   ↓
 模式选择页    随性漫游 / 深层剖视
   ↓
/chat         对话散步页（核心触点）
/map          画像页（19 维展示 + 证据层）
/collision    关系碰撞页（三种模式）
/timeline     时间轴页（演化追踪）
```

### 3.2 对话系统（ChatPage）

**核心设计原则：** 去模板化。AI 不从题库出发，从用户当前表达出发。

| 机制 | 说明 |
|------|------|
| 回声机制 | 每 2-3 轮用自己语言重新表述用户核心逻辑 |
| 矛盾捕捉 | 行为矛盾 > 情境矛盾 > 价值矛盾，三层递进 |
| 洞察给出 | 命名型 → 矛盾揭示型 → 呈现型，不按轮次强制 |
| 收束逻辑 | 12 维强信号自然收束，19 维全触及主动问 |
| 随时退出 | 进度条展开面板 ≥6 维即可查看结果 |

**深层剖视模式专属：** 四重思维框架内化到对话中。

| 框架 | Oprah 的呈现 | 不说的话 |
|------|-------------|---------|
| 苏格拉底诘问 | "你为什么觉得是这样的？如果反过来呢？" | "我现在用诘问法" |
| 第一性原理 | "这个想法是从哪里来的？也许它不是你的声音" | "回到第一性原理" |
| 布鲁姆金字塔 | 收束时从行为→动机→可能性，拼三维图 | "认知层级" |
| 水平思考 | "我们换个角度？" → 给出第三个选项 | "水平思考" |

### 3.3 画像系统（MapPage）

- **19 维人格模型：** 思维 5 维 + 价值观 5 维 + 关系模式 5 维 + 未完成自我 4 维
- **三级确信度：** 已探索（70-90）/ 推测（40-65）/ 未探索（20-35），UI 灰显区分
- **证据层：** 摘要 → 引用来源 → 反例检查，三层层层展开
- **"我不认同"出口：** 方向调整 / 重新探索 / 完全重来，三条修正路径

### 3.4 关系碰撞（CollisionPage）

三种碰撞模式：

| 模式 | 触发条件 | 精度 |
|------|---------|------|
| 双人完整碰撞 | 双方都完成画像 | 高 |
| 单人推测碰撞 | 暗号不存在 → 用户描述朋友 | 中（标注推测） |
| 原型碰撞 | 12 个预置角色，单人可玩 | 中（基于原型） |

碰撞结果按关系类型差异化展示优先级：
- 恋人 → 依恋兼容性优先
- 密友 → 共鸣区优先
- 家人 → 边界冲突优先
- 同事 → 工作风格差异优先

### 3.5 时间轴（TimelinePage）

- 身份标签演化时间线
- 跨关系重复模式检测（如"你在不同关系中反复成为'磨脑搭子'"）
- 每周反思触达（基于当前身份标签动态生成）
- 7 天记忆整理仪式入口（视觉已设计，逻辑待实现）

## 四、人格标签体系

### 4.1 设计原则

从学术长词 → 2-4 字热梗短词。像 MBTI 的 INFJ 一样可以轻松说出口。

### 4.2 标签三件套

| 类别 | 数量 | 用途 | 示例 |
|------|------|------|------|
| 身份标签 | 15 个 | 结果页展示，一句话说清"我是什么人" | 和事佬、守界人、架构师、捕手 |
| 碰撞角色 | 28 个 | 关系双方互相的定位 | 磨脑搭子、避难所、兜底人、照妖镜 |
| 关系类型 | 10 个 | 碰撞结果顶层命名 | 世另我、天生绝配、越吵越铁、安心搭子 |

### 4.3 标签生成逻辑

规则+LLM 混合：按维度组合匹配标签库，库无匹配时 LLM 创造（≤10 字）。

## 五、技术架构决策

### 5.1 为什么选这个技术栈

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | React + Vite + TypeScript | 快速迭代，生态成熟 |
| 样式 | Tailwind CSS + 自定义 CSS | 组件级样式 + 全局主题变量 |
| 数据层 | Supabase（备选 localStorage） | 无需后端，JSONB 灵活，RLS 安全 |
| LLM 代理 | Cloudflare Pages Functions | API key 不暴露到前端 |
| LLM 路由 | TokenRouter → Claude Haiku 4.5 | 成本可控，延迟低 |
| 部署 | Cloudflare Pages | 免费，全球 CDN，Functions 支持 |
| 状态管理 | React Context | 够用，不引入 Redux 复杂度 |

### 5.2 数据架构

```
users 表
├─ pin_code (TEXT PK)          # 6 位数字
├─ match_code (TEXT UNIQUE)    # 4 位大写字母
├─ chat_history (JSONB)        # ChatMessage[]
├─ dimensions (JSONB)          # DimensionResult
├─ progress (JSONB)            # ProgressData
├─ identity_label (JSONB)      # IdentityLabel
├─ analysis_versions (JSONB)   # AnalysisVersion[]
├─ created_at / updated_at

collisions 表
├─ id (UUID PK)
├─ user_pin (TEXT FK)
├─ friend_code (TEXT)
├─ result (JSONB)              # CollisionResult
├─ relationship_context (TEXT) # romantic/close_friend/...
├─ created_at
```

**为什么用 JSONB：** 人格模型、标签体系、证据结构会持续迭代。JSONB 允许 schema 变化不迁移数据库。

### 5.3 LLM 调用架构

```
浏览器 fetch('/api/llm')
  → Cloudflare Function (functions/api/llm.ts)
    → 从 Secret 读取 TOKENROUTER_API_KEY
    → POST https://api.tokenrouter.com/v1/chat/completions
    → Claude Haiku 4.5 响应
    → 返回前端
```

前端代码中永远不会出现 API key。

| 要素 | 值 |

## 七、增长与传播

### 7.1 自然传播路径

```
用户完成画像 → 获得身份标签（如「架构师」）
  → 和朋友碰撞 → 获得关系命名（如「世另我」）
    → 保存关系名片（Canvas 1080×1920）
      → 发朋友圈/小红书/微信群
        → 新用户扫码进入
```

### 7.2 传播资产

| 资产 | 形式 | 目的 |
|------|------|------|
| 身份标签 | 2-4 字短词 | 像 MBTI 一样可口头传播 |
| 关系名片 | 1080×1920 图片 | 朋友圈/小红书分享 |
| 关系命名 | 10 个热梗词 | "我们居然是世另我" |
| 暗号 | 4 位大写字母 | 朋友输入即可碰撞 |

### 7.3 冷启动策略

- **原型碰撞：** 12 个预置角色，单人即可体验完整碰撞流程，不依赖朋友在系统内
- **推测碰撞：** 朋友不在系统内时，用户描述朋友即可生成碰撞 + 邀请卡
- **双版本部署：** 不同风格的版本覆盖不同审美偏好的用户群

## 九、不足与路线图

### 9.1 当前不足

| 问题 | 严重程度 | 方向 |
|------|---------|------|
| UI 设计简陋 | 高 | 学习 AI+素材组合，自适应风格生成 |
| 进入页缺吸引力 | 高 | 粒子动画+锋利文案制造好奇心 |
| 社交符号未建立 | 高 | 与星座/MBTI/玛雅历互译 |
| 情感依附弱 | 中 | "被翻阅的成长日记" |
| 交互形式单一 | 中 | 语音→视频路线图 |
| 对话仍有机械感 | 中 | 回答长度自然化，新旧用户差异深化 |
| 心理学基础弱 | 高 | 嵌入依恋/正念/存在主义框架 |
| 无听觉维度 | 低 | BGM：莫扎特/贝多芬/巴赫匹配人格 |
| 缺轻量场景 | 高 | 心情日记：30 秒日常+心情曲线 |
| 无社会议题回应 | 中 | ADHD/内耗等焦虑话题温和视角 |

### 9.2 短期路线图（接下来两周）

```
Week 1:
  ├─ UI 全面提升（学习 AI 设计工具 + 素材库）
  ├─ 进入页动态视觉钩子
  └─ 心理学框架系统化嵌入 prompt

Week 2:
  ├─ 社交符号体系（星座/MBTI 互译）
  ├─ 对话去模板深化
  └─ 心情日记 MVP（一句话记录+心情曲线）
```

### 9.3 中期方向

- 自适应 UI：根据对话中用户审美偏好，动态调整界面风格
- 语音输入 + AI 形象回应
- 人格星盘/能量值等社交传播符号
- BGM 系统（古典乐人格匹配）
- 追踪流行心理话题的温和回应能力
